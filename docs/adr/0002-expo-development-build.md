# 0002. Mobile stack: Expo development build

- Status: accepted

## Context

Legacy mobile was a bare React Native 0.72 app. Restarting maintenance needs a supported toolchain, while Discovery/Session still require custom native UDP/TCP modules that Expo Go cannot ship.

## Decision

Migrate to Expo SDK with a local development build workflow:

- `expo-dev-client` + local `expo prebuild` / `expo run:android` (or `expo run:ios` on macOS)
- Keep `react-native-tcp-socket` and `react-native-udp` via prebuild / custom native build
- Prefer Expo modules for file, clipboard, audio, notifications, network where they replace legacy libs
- UI keeps RNEUI where peer deps allow; otherwise RN core + `@expo/vector-icons`
- Retain MobX store shape for parity with legacy connection flow
- Packaging is local only; CI uses GitHub Actions (no EAS)

## Consequences

- Cannot rely on Expo Go for full feature testing; use local development builds.
- Android/iOS native projects are generated by prebuild (CNG) unless ejected intentionally.
- Release artifacts are produced on maintainer machines or GitHub Actions runners, not Expo Application Services.
