# Syncer Design

局域网协作工具的视觉与交互方向。两端保持同一品牌信号，桌面端偏 Ant Design 密度，移动端偏原生控件简洁布局。

## Brand

- **产品名**：Syncer，在窗口标题 / 应用名中直接出现，不使用副品牌口号压过名称。
- **品牌色**：`#fa8c16`（Ant Design orange-6），用于主按钮、强调图标、分段控件选中态。
- **悬停 / 按下**：`#ffa940` / `#d46b08`。
- **避免**：紫系渐变、深色默认主题、大面积发光、圆角胶囊标签堆叠。

## Typography & density

- 基础字号约 14px；设备名用更大字重作为页面主标题。
- 次要信息（IP、提示文案）使用降低透明度的黑色（约 40%–65%）。
- 桌面端页面内边距约 32px（连接页局部可收至 16px）；移动端保持紧凑列表与清晰触控目标。

## Layout & navigation

- **非仪表盘**：首屏只有「本机身份 + 发现/连接」或「已连接后的发送区」一种主任务。
- **状态驱动视图**：`available` / `connecting` → 连接页；`connected` → 发送页。不使用多级导航栏。
- **连接中**：模态等待，可取消；不在后台静默转圈。
- **连接失败**：超时、不可达、对端忙和协议不匹配都应给出明确反馈，然后回到可再次 Discovery 和连接的状态。

## Components

- 桌面：Ant Design Vue（按钮、输入、卡片、分段控件、上传、模态）。
- 移动：React Native 核心组件 + `@expo/vector-icons`；主色与桌面一致。
- **卡片**：仅用于 Available Device 列表项（可点击交互容器）；发送页不用装饰性卡片。
- **模态**：连接请求、收到文本/文件、查找设备等打断性确认；内容区限制高度并可滚动。

## Interaction

- Discovery：默认广播查找；支持手动输入 IP。
- Session 内用分段控件切换 Text / File / Command（或 Find Device）。
- 白名单以复选框呈现：「自动接受此设备的连接请求」。
- 接收历史为次级入口（图标按钮），分页加载，支持多选删除。

## Motion

- 保持克制：加载态用按钮 loading / Spin；列表出现无需夸张动画。
- 查找设备时移动端以持续振动 + 铃声作为强反馈（功能反馈，非装饰动效）。

## Platform notes

- 桌面：关闭窗口隐藏到托盘，退出经托盘菜单。
- 移动：需允许后台运行才能稳定接收 Connection Request（厂商省电策略见 README）。
